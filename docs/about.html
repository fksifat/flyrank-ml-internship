Plain answer

Static HTML, shared CSS, relative paths, GitHub Pages.

There is no framework here. The portfolio is a small static site served from the repo’s docs/ folder, with each page linked by hand so every page is obvious and inspectable.

Stack

Markup

HTML pages in docs/ for the site structure and content.

Style

One shared CSS file with responsive layout rules and a warm visual palette.

Hosting

GitHub Pages serves the repository directly, so the public URL stays tied to the repo.

What I can explain

  • Why the pages are separate: every case opens on its own URL and is reachable from the nav.
  • Why the assets are local: SVGs render cleanly, stay light, and work with relative paths.
  • Why the tone is plain: the goal is to show the work, not hide it behind fancy polish.
