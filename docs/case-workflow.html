Automation workflow v2

NotebookLM to Claude Project to custom GPT to human review.

This workflow turns a weekly research packet into a repeatable brief. It is intentionally not fully autonomous; the final human review is part of the design, not a bug.

Back to home How the site is built

What the workflow does

Step 1

NotebookLM collects source notes

Start with the source packet so the brief is grounded in the same inputs every week.

Step 2

Claude Project pressure-tests the draft

The custom instruction is to challenge fluff, keep the structure tight, and point out missing evidence.

Step 3

Custom GPT shapes the final wording

This is where the tone is made consistent and the draft becomes easy to post or share.

Step 4

Human review approves the output

The last pass is manual because the summary should still reflect judgment, not just automation.

What I would explain if asked

  • The workflow is fixed, so anyone can rerun it without hidden branching logic.
  • Every step has a role: gather, critique, draft, review.
  • The failure points are obvious: bad source notes, overlong draft prompts, or skipping human review.
  • The current timing worked out to a 26-minute net save after setup, which is enough to justify using it again.

Still rough

It is good enough to show the mechanics, but it still needs a cleaner visual asset, a tighter example prompt, and a better final handoff format.
