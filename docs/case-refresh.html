What I built

Method

Logistic regression for a readable anchor, a decision tree for a simple nonlinear check, and a random forest as the strongest candidate.

Split

Client-holdout split with 26 training clients and 6 held-out clients. That keeps client patterns from leaking across the validation boundary.

How to read the errors

False positives

The model can overcall decline on visible pages that are still healthy enough to hold up, especially when the position and age signals look stale but the content is not actually slipping.

False negatives

It misses some low-visibility pages with very little behavioral signal. That is a useful boundary: the model is stronger when there is enough history to read.
